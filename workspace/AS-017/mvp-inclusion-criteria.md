# MVP Inclusion Criteria

> Rascunho AS-017 — NR-002.

---

## Critérios Must (todos)

1. Necessário para **Institution Care Journey** ambulatorial (+ tele como mode)
2. Necessário para **segurança multi-tenant** mínima (Identity/AuthZ/Audit/Tenant)
3. Necessário para **Registrar** o cuidado (documentação/ordens/artefatos básicos)
4. Não depende de domínio Extension ou Hospital Future
5. Não é Billing/estoque/financeiro profundo (Q-005)

## Tiers

| Tier | Significado |
|---|---|
| **MVP-Must** | Entra na primeira fatia implementável |
| **MVP-Should** | Logo após Must; não bloqueia validação clínica staff |
| **Later** | Catálogo permanece; fora do MVP inicial |
| **Out** | Explicitamente fora até Q-005 / Future (Billing, Hospital, etc.) |
